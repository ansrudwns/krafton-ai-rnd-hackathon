# SparseTap 챌린지 기술 보고서 (Technical Report)

## 1. 문제 분석 및 수식화
SparseTap 챌린지는 선형 피드백 시프트 레지스터(LFSR) 메커니즘을 통해 256비트 시퀀스를 생성하는 데 사용된 숨겨진 $S \le 16$ 개의 탭 오프셋(Offsets) $d_1, \dots, d_S \le 64$ 를 식별하는 문제입니다. 관측 방정식은 다음과 같이 정의됩니다:
$$y_n = \bigoplus_{j=1}^S y_{n-d_j} \oplus e_n$$
여기서 $y$는 관측된 비트 시퀀스, $e_n \sim \text{Bernoulli}(0.2)$는 $20\%$ 확률의 노이즈를 의미합니다. 동일한 탭 구성을 가졌지만 랜덤한 시드로 생성된 2,000개의 독립적이고 노이즈가 섞인 시퀀스를 바탕으로 숨겨진 오프셋을 찾는 것은 암호학에서 난제로 알려진 **LPN(Learning Parity with Noise)** 문제를 대수적으로 푸는 것과 완전히 동일합니다.

## 2. 실패한 접근 방식: 단순 교차 상관 분석 (Cross-Correlation)
초기에는 관측된 값 $y_n$과 과거 비트들 $y_{n-k}$ 간의 단순 교차 상관(Cross-correlation)을 계산하는 접근을 시도하여, 특정 오프셋에서 통계적으로 유의미한 상관관계가 나타날 것이라 가정했습니다. 하지만 LFSR 구조 특성상 시퀀스가 강한 의사 난수(Pseudo-random) 특성을 띠기 때문에 단일 과거 비트와의 상관관계는 0으로 상쇄(50% 확률 유지)되어 **완전히 실패**했습니다. 상관관계는 숨겨진 모든 비트 $S$ 개를 동시에 결합(Joint Parity)해서 평가할 때만 나타나므로, 단순 통계적 지연(Lag) 연산으로는 개별 오프셋들을 분리해 낼 수 없습니다.

## 3. 방법론 1: 연속 완화 및 경사하강법 (Method 1: Numerical/Gradient Descent)
챌린지 요구사항인 수치적 방법(예: Gradient Descent)을 통해 이산 공간의 부울(Boolean) 다항 마스크를 검출하기 위해, 불연속적인 XOR 제약 조건을 부드럽게 미분 가능한 손실 함수(Loss function)로 재구성하는 **연속 완화(Continuous Relaxation)** 기법을 적용했습니다.

**도메인 변환:**
우리의 목표는 $B \equiv A \cdot c \pmod 2$ 식을 만족시키는 실수 벡터를 구하는 것입니다. XOR 연산을 곱셈 성질로 변환하기 위해 $0, 1 \to +1, -1$ 매핑을 사용할 수도 있지만, 이보다 더 간결하게는 $\cos(\pi(B - A \cdot c))$ 식을 활용할 수 있습니다. 방정식이 참일 경우 해당 수식의 괄호 내부는 항상 짝수 정수가 되어 코사인 값이 $1$이 되며, 오차가 있을 경우 홀수 정수가 되어 $-1$이 산출됩니다.

**최적화 목표 (Optimization Objective):**
우리는 미분 가능한 매개변수 벡터 $c \in \mathbb{R}^{64}$ 를 선언하고, Parity 일치 확률을 최대로 높일 수 있는 "음의 기대 코사인 척도"와 더불어, 명확한 희소성($S \le 16$)을 강제하기 위한 L1 정규화 항을 결합하여 최소화했습니다:
$$L(c) = - \frac{1}{M}\sum_{i=1}^M \cos \left( \pi \left( B_i - \sum_{j=1}^{64} c_j A_{i,j} \right) \right) + \lambda \|c\|_1$$
위의 손실 함수는 부드러운 곡선(Smooth Surface)을 가져 그레이디언트(Gradient)가 전파되게 합니다. 프레임워크 병목을 차단하기 위해 손실 함수의 편미분 수식인 $\nabla_c L(c) = -\frac{\pi}{M} \sum \sin(\pi(B_i - A_i \cdot c))A_i$ 를 직접 산출하고 Numpy 고속 벡터 연산만으로 Adam 알고리즘을 자체 구현했습니다. 384,000개의 샘플에 대해 최적화를 진행한 결과 벡터의 크기 $|c_j|$ 가 숨겨진 오프셋 위치에서 정확히 최고치를 형성하는 것을 확인했습니다.


## 4. 방법론 2: 가우스 소거 기반 고속 탐색 (Method 2: Information Set Decoding)
얻어낸 해답을 대수적으로 한 번 더 교차 검증하기 위해, LPN 해법에 맞춤화된 대표적인 정보 집합 디코딩(Information Set Decoding, ISD) 방식을 사용했습니다.
노이즈 비율이 $20\%$ 이므로, 우리가 가진 방정식의 약 $80\%$는 노이즈가 전파되지 않은 완전히 깨끗한 $A \cdot x = B \pmod 2$ 상태의 식입니다.

**확률적 가우스 소거법 (Probabilistic Gaussian Elimination):**
무작위로 샘플링한 64개의 방정식이 모조리 깨끗할 확률은 $0.8^{64} \approx 6.27 \times 10^{-7}$ 입니다. 총 384,000개라는 압도적인 수량의 방정식이 주어졌기 때문에 반복적으로 64개의 행을 추출하여 초고속 GF(2) XOR 기반 가우스 소거법을 적용해 역행렬을 도출하고, 구해낸 $x$ 후보를 별도의 1,000개 방정식에 대입하여 $\approx 80\%$ 일치율에 도달하는지 여부로 진정한 해답임을 검증했습니다.

동시성과 비트 단위 최적화 기법을 도입한 우리의 대수적 Python 역행렬 탐색 모델은 표준 CPU 환경에서 단 몇 초 만에 수백만 개의 행렬을 연산해 해답을 찾았습니다. 그 결과로 얻어낸 LFSR 오프셋 값들이 앞선 1번 수치적 방법(Gradient Descent)의 결론과 정확하게 일치하며 양방향 검증에 완전히 성공했습니다.
