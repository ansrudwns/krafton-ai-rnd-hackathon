## 🎯 예선 과제 1: SparseTap (비선형 암호 해독 및 LPN 최적화)

### 1. 문제 정의 (Problem Statement)
- **목표:** 특정 선형 피드백 시프트 레지스터(LFSR)를 통해 생성된 256비트 시퀀스 중에서, 노이즈가 섞인 2,000개의 관측치로부터 16개 이하의 숨겨진 오프셋(Offsets)을 식별하는 문제.
- **특징:** 암호학 난제인 **LPN(Learning Parity with Noise)** 문제를 대수적 및 수치적으로 해석해야 함.

### 2. 나의 풀이 (Methodology)
통계적 교차 상관(Cross-Correlation) 분석 한계를 인지한 후, 두 가지 상호 보완적인 파이프라인을 구축하여 해답을 교차 검증했습니다.

1. **연속 완화 및 경사하강법 (Continuous Relaxation via Gradient Descent):**
   - 불연속적인 이산 XOR 제약 조건을 코사인 변환을 통해 미분 가능한 '음의 기대 코사인(Negative Expected Cosine)' 손실 함수로 변환했습니다.
   - 희소성 제약 보강을 위해 L1 정규화를 결합하고, Numpy 기반 고속 전파(Adam Opt.)를 통해 오프셋 위치를 수치적으로 탐색했습니다.
2. **정보 집합 디코딩 기반 고속 탐색 (ISD via Gaussian Elimination):**
   - 데이터 관측치에 80% 이상의 상대적으로 노이즈가 없는 원본 데이터가 존재함을 이용하여 확률적 가우스 소거법을 설계했습니다.
   - GF(2) XOR 기반 역행렬 연산 모델을 구축하여, 샘플링된 수식의 역행렬을 수학 공간에서 검증하고 최종 결과를 안정적으로 도출했습니다.

## 🎯 예선 과제 2: MultiplierBoard (Transformer 설계 축소 한계 분석)

### 1. 문제 정의 (Problem Statement)
- **목표:** 트랜스포머 아키텍처(Self-Attention) 단일 모델을 사용하여 자기회귀(Autoregressive) 이진 곱셈을 수행하는 **최소 파라미터(Minimal Parameter)** 모델 성능 분석.

### 2. 나의 풀이 (Methodology & Architecture)
1. **최소 레이어 계층 증명:**
   - 구조적으로 1-Layer 트랜스포머는 비트 부분곱 합산 연산과 지연된 올림수(Carry)의 라우팅을 동시에 병렬 처리할 수 없음을 수식으로 확인했습니다.
   - 이에 따라 연산 과정을 해결할 수 있는 **2-Layer, 2-Head Attention 모델**이 알고리즘적으로 필수적임을 도출했습니다.
2. **파라미터 최적화 (Weight-Tying):**
   - 규칙 내에서 Token Embedding과 Output Projection 가중치를 연결하는 **가중치 공유(Weight Tying)** 기법을 통해 파라미터 사용을 효율적으로 절감했습니다.
   - **이론적 최소치와 학습 최적화의 간극 규명:** 이론상 최소 파라미터 구조(12차원, 4,226개)는 최적화 과정에서 Gradient Bottleneck이 발생하여 수렴율이 크게 제한(5.5%)되는 현상을 확인했습니다.
   - 학습 표현력을 부여하기 위해 히든 스페이스를 늘린 16차원(6,594개 파라미터) 아키텍처로 튜닝하여 99.5% 이상의 모델 수렴을 달성했습니다.

## 🎯 본/결선 DAY 1: BattlePredict (배틀로얄 시계열 예측과 수리적 복원)

### 1. 문제 정의 (Problem Statement)
- **목표:** 10명의 플레이어가 50일간 진행하는 배틀로얄 경기 데이터(1v1 건틀릿 릴레이 매치) 중, 데이터가 은닉된 **나머지 29일(Day 22~50)** 간의 플레이어별 누적 킬 수를 예측
- **제약 조건:** 분석 대상 기간 동안 발생하는 총 킬 수는 **5800킬**(1개 매치 당 4킬)로 고정되어 있으며, 10명의 예측 총합이 이 수치 체인과 구조적으로 일치해야 함.
- **데이터 특성:** 비추이적(Non-transitive) 상성 모델과, 시간에 따라 성능 편차가 일어나는 **메타 시프트(Temporal Drift)**가 내포됨.

### 2. 현장 제출용 구동 구조 (Initial Solution & Methodology)
결선 현장에서 시계열 데이터를 분석하고 $P(i > j)$ 확률망을 추론하기 위해 다음과 같은 예측 파이프라인을 구축했습니다.

1. **Hybrid Probability Matrix (Bradley-Terry + Empirical Counts):**
   - **Bradley-Terry Prior:** 1v1 대전 기록 결측치를 보완하기 위해 최우도 추정(NLL) 기반 BT 모델을 적용, 선수별 잠재 실력 점수(Latent Skill Score)를 도출했습니다.
   - **승률 결합:** 대전 쌍의 비추이적 상성을 시스템적으로 반영하기 위해 BT 맵핑 확률과 관측망 승률망(Empirical Matrix)을 상수 비율 가중치($\alpha$)로 결합했습니다.
   - **지수 감쇠 필터 (Exponential Time Decay):** 매치의 타임라인 흐름에 대응하기 위해 최신 메타 데이터를 중용하도록 $\exp(-\lambda \frac{N-1-i}{N})$ 기반의 감쇠율을 그래디언트에 적용했습니다.
2. **Markov Chain 기댓값을 활용한 체인 정합성 보완:**
   - 연속된 5인 건틀릿에서 늦게 입장하는 유저의 포지션 불균형(Position Effect)을 해결하기 위해 확률 전이 구조인 **마코프 체인(Markov Chain)**을 활용했습니다. 
   - 결과적으로, 시뮬레이션을 쓰지 않고도 매 건틀릿 매치별 누적 킬수의 기댓값 합산이 **4.0**으로 보존되어 총점 '5800점'의 제약을 사후 처리 과정 없이 모델링 내부에서 해결하는 수리 구조를 확보했습니다.

### 3. 사후 분석 및 파라미터 튜닝 개선 (Final After / Post-mortem)
기초 수학적 모델링은 동작했으나 현장 테스트 시 하이퍼파라미터 튜닝 과정에 결함이 있었습니다. 사후 검토 중 Validation 분할 방식과 정규화 스케일링에서 큰 편향 구조를 찾아 개선했습니다.

1. **장기 척도 검증 (Long-Horizon Validation) 단위 수정:**
   - **현상 파악:** 현장 최적화 진행 시, `Day 1~20 학습 -> Day 21 (1-Step Ahead) 예측`이라는 단기적 검증에 과적합되어, $\lambda$(시간 감쇠 계수) 파라미터가 장기적 스케일을 무시하고 단기 폼에만 편향 반영되도록 편중(Under-smoothed)됨.
   - **개선안 (Final After):** Validation Split을 `Day 1~11 학습 -> Day 12~21 기간 장기 평가` 구조로 조정. 한 달 이상의 예측 범위에 필요한 장기 분포(Long-Tail) 특징을 학습 내에 정착시켰고 과적합 현상을 유의미하게 제한했습니다.
2. **동적 그래디언트의 L2 정규화 (Dynamic L2 Penalty Scaling) 적용:**
   - **현상 파악:** 시간 감쇠를 거치면 후속 그래디언트의 유효 질량(Effective Sample Size)이 급감함에도, L2 규제가 단일 고정 상수(`1.0`)로 매핑되어 모델 파라미터가 지나치게 억제되는 언더피팅(Underfitting) 발생.
   - **개선안 (Final After):** 시간 감쇠 변화량에 대응해 L2 규제 상수를 동적으로 조절하는 **비례 스케일링** 로직을 도입.
     ```python
     sum_weights = sum(math.exp(-lambda_val * (N - 1 - idx) / N) for idx in range(N))
     l2_lambda = 0.5 * (sum_weights / N) 
     skills[i] += lr * (grads[i] - l2_lambda * skills[i])
     ```
   - 그래디언트 위축분과 페널티 위축분을 독립적(Decouple)으로 제어하며 모델이 갖춰야 할 본래의 적정 예측 성능을 안정적으로 활성화했습니다.

## 🎯 본/결선 DAY 2: VideoAgent (이기종 멀티모달 비디오 분석 과제)

### 1. 문제 정의 (Problem Statement)
- **목표:** 최대 20분 길이 거대 비디오 쌍 20개에서 발생하는 사건에 대한 비전 질의응답 및 객관식 정답 분석 도출.
- **제약 조건:** 시스템 실행부터 모델 처리 후 결과 제출까지 **제한 시간 15분 이내**에 파이프라인 구동을 안정적으로 수행해야 함.

### 2. 현장 구축 모델과 병목 원인 진단 (Initial Solution & What Went Wrong)
1. **시-공간 압축 샘플링:** 통신 오버헤드를 막기 위해 임베디드 Subprocess에서 `FFmpeg` 쉘을 호출하여 640px 대역 및 초당 1프레임 환경으로 강제 다운샘플링 진행.
2. **이기종 맵리듀스망:** 가벼운 인지 워커 모델(`Gemini Flash`)이 타겟 도메인을 분석해 텍스트 로그 단위로 변환하고, 추론 모델(`Gemini Pro`)이 정보 병합 후 질의응답.
3. **현장 실패 분석:** 다중 API 키를 Round-Robin으로 분산하려 했으나 비동기 I/O 에러가 발생하여 응답 지연 현상을 겪었습니다.
   - **PROCESSING 무한 대기:** 외부 손상 청크에서 서버 상태 락(Lock)이 해제되지 않아 `time.sleep()` 자원 데드락 발생.
   - **태스크 누수 (Task Leak):** `asyncio` 병렬화 시 예외(Exception) 전파 발생 후 Sibling 프로세스의 통제가 상실되어 시스템의 트래픽을 백그라운드 형태로 점유.
   - **Dead Key 병합 오류:** Rate Limit 초과 시 지연 시간(Backoff)은 가져갔으나 새로운 환경 키로의 스와핑이 되지 않아 전체 레이턴시 오버헤드 유발.

### 3. 사후 통합 아키텍처 개편 및 클라우드 파이프라인 고도화 (Final After)
대회 종료 후 로컬 기반 추론 서버의 지연을 해결하기 위해 Vertex AI 환경에 맞춘 파이프라인 전면 이관을 단행, 병목 구간을 제거했습니다.

1. **Vertex AI 백엔드 통합 연동:**
   - 구조적으로 불안정했던 외부 계정 키 분할 로직을 철회하고 **구글 클라우드의 커머셜 `google.genai` SDK 풀 환경**으로 이관하여 시스템 자원 관리 안정성 한계를 돌파했습니다.
   - 비동기 에러 중 잔여 청크들이 메모리를 오버로드 하지 않도록 `finally` 단 내부 `glob.glob` 명령어를 통해 파일 찌꺼기와 워커 시스템 종료를 클린 업 수행. 
2. **토큰 절약에 따른 해상도 스케일업 파이프라인 구축 (720px / 2fps):**
   - 백엔드의 통신 설정에 `media_resolution="MEDIA_RESOLUTION_LOW"`를 삽입, 프레임 당 토큰 발생률을 고정 70토큰 수준으로 억제했습니다.
   - 트래픽 자원 병목이 크게 해결되며, 원천 입력 영상 품질을 오히려 상향(`720px/2fps`) 조절시켜 인지 워커 모델의 비전 디텍팅 정확도를 끌어올렸습니다.
3. **인지 타겟 제어 및 (Worker Target Bias) 기법 활용:**
   - 인지 워커의 환각 응답(Hallucination) 방지를 위해 정규식을 이용해 메인 스크립트에서 객관식의 조건 값 범위(예: "A)11개")를 분석했습니다. 이 범위를 워커의 프롬프트인 `TARGET BIAS` 블록에 제한 값으로 주입하여, 전체 프레임 중 특정 범위 값의 데이터만 집중적으로 축출 탐색하도록 유도했습니다.
4. **마스터 모델 체인 프롬프트 (Self-Reflection):**
   - 통합 질의 응답 모델에 **2-Step Prompt Chain** 방식을 사용, 논리 오류를 자체 점검하게 구성했습니다.
   - 1차적으로 로그에 대한 소거법 기반 임시 추론을 내리게 한 뒤, *"넌 비평가다. 더블 카운팅이나 산술 착오 여부를 스스로 교차 점검하라"* 는 지시어를 구성, 이후 XML 형태(`<ANSWER> A </ANSWER>`) 객관식으로 결과를 필터링했습니다.
5. **결론 (Final Result):** 
   - 아키텍처 개편 및 구조화 프롬프트 튜닝 결과 환각 및 답변 포맷 유실이 크게 통제되었습니다. 
   - 다중 스레드 환경 검증 데이터 지표를 살펴본 결과, 20문항의 정답 출력 문자열이 **`CGPCDMDOFFNBPIBDCDKF` (315.6초)** 와 **`CGPCDMDOFFNBPIBDCDMF` (666.1초)** 로 기록되어, 1문항에서만 오차가 발생하는 수준으로 **추론의 일관성이 매우 단단하게 상승**함을 확인했습니다. 
   - 더 이상 데드락에 빠지지 않고, 15분 타임어택보다 훨씬 빠른 수 분 이내로 20개의 처리 연산을 안정적으로 마감했습니다.
